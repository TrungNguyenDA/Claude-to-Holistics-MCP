Connecting Claude to Holistics via MCP

Query and explore Flodesk's business data through natural conversation with Claude — no SQL required.

Overview

This guide walks you through connecting Claude Desktop or Claude.ai to Holistics via MCP (Model Context Protocol).

⚠️
PII data (emails, names, etc.) is intentionally masked in Holistics. If you need to access PII data, please use the Metabase self-serve tool instead.

Step 1 — Request a Holistics API Key

Before you can connect Claude to Holistics, you need a personal Holistics API Key.

1

Create a request ticket

Open the Holistics MCP project on Linear and create a new issue to request API key enablement.

2

Include your details

Add your name and team in the ticket title so the Data Team can identify and route your request.

3

Wait for follow-up

Trung or Hang will follow up to enable your access and guide you to generate your personal API key.

🔐
Your API key is personal and non-transferable. You are responsible for keeping it secure. Do not share it with others, commit it to any codebase, or store it in a shared document. If your key is compromised, contact Trung or Hang immediately to revoke and reissue it.

Step 2 — Connect Claude to Holistics

Choose your preferred Claude client below.

1

Open the config file

In Claude Desktop, go to Settings → Developer → Edit Config. This opens the file claude_desktop_config.json in your text editor.

2

Add the Holistics MCP server

If the file is empty, paste the full config below. If it already has content, add the "holistics" block inside the existing "mcpServers" object.

Empty file — paste this entirely:

claude_desktop_config.json 
{
  "mcpServers": {
    "holistics": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://mcp-us.holistics.io/reporting/mcp",
        "--header",
        "X-Holistics-Key:YOUR_API_KEY_HERE"
      ]
    }
  }
}

File already has content — add only the holistics block:

claude_desktop_config.json 
{
  "mcpServers": {
    // ... your existing servers ...
    "holistics": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://mcp-us.holistics.io/reporting/mcp",
        "--header",
        "X-Holistics-Key:YOUR_API_KEY_HERE"
      ]
    }
  }
}
⚠️
Replace YOUR_API_KEY_HERE with your actual API key. Do not add a space after X-Holistics-Key:
3

Restart Claude Desktop

Save the file, then fully quit Claude Desktop (Cmd + Q on macOS) and reopen it. A simple window close won't reload the config.

1

Open connector settings

Go to claude.ai/settings/connectors and click Add custom connector.

2

Enter the Holistics MCP details

FieldValue
NameHolistics
URLhttps://mcp-us.holistics.io/reporting/mcp
3

Authenticate

Click Add, then Connect and follow the OAuth flow to authenticate with your Holistics account. No API key needed for this method.

4

Enable in chat

In any chat, click the tools icon (left of the message field) and enable Holistics under Connectors.

Step 3 — Verify the Connection

Confirm everything is working with a quick test.

✓

Test it out

Open a new Claude conversation and ask:

Prompt 
Check connection to Holistics

If the connection is working, Claude will confirm the connection and show you available datasets.

🎉
Once verified, you're all set! You can now ask Claude questions about Flodesk's business data in natural language.

API Key Responsibilities

By requesting and using a Holistics API key, you agree to the following.

  • Keep your API key private and secure
  • Use it only for legitimate work purposes at Flodesk
  • Do not share your API key with colleagues — each person must have their own
  • Do not use your key to extract or export data beyond your role's permitted access
🛑
Flodesk's Data Team reserves the right to revoke any API key at any time if misuse is detected.

Common issues and how to resolve them.
